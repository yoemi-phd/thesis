Gabarit LaTeX pour mémoires et thèses / LaTeX template for dissertations and theses


1) Introduction

    Pour Linux, un Makefile est distribué afin d'automatiser le processus de recompilation. Toutefois, ce Makefile n'est offert qu'à titre d'exemple et peut devoir être modifié, selon la distribution de Linux utilisée.

    Le processus de compilation permet de regénérer les graphiques et images produits à l'aide des logiciels Gnuplot et Dia. Il est préférable que toutes les images importées soient en format EPS (Encapsulated Postscript) ou en tout autre format d'image vectoriel. D'autres types d'images peuvent être supportés, selon la version de LaTeX.

    For Linux, a Makefile is distributed to automate the recompilation process. However, this Makefile is only offered as an example and may have to be modified, depending on the Linux distribution used.

    The compilation process allows to regenerate the graphics and images produced with Gnuplot and Dia. It is preferable that all imported images be in EPS (Encapsulated Postscript) format or any other vector image format. Other types of images may be supported, depending on the LaTeX version.

4) Le document LaTeX / The LaTeX document

   Il existe deux modes ("draft" et "final") pour le document LaTeX. Le mode "draft" génère un fichier DVI sans images, mais réserve l'espace pour ces dernières dans le texte. Le mode "draft" permet aussi de reconnaître facilement les débordements en fin de ligne car un carré noir est ajouté à la ligne fautive.

   Le mode "final" permet de générer la version finale d'un document. Il inclut toutes les images mais n'identifie pas les lignes qui débordent dans la marge de droite. Vous devriez toujours lire les messages affichés par LaTeX afin de corriger les erreurs.

   There are two modes (draft and final) for the LaTeX document. The draft mode generates a DVI file without images, but reserves space for them in the text. The draft mode also makes it easy to recognize overflows at the end of a line because a black square is added to the faulty line.

   The final mode generates the final version of a document. It includes all images but does not identify lines that overflow into the right margin. You should always read the messages displayed by LaTeX to correct errors.

   Lors d'un débordement en fin de ligne causé par un mot que LaTeX ne reconnaît pas, il est possible d'indiquer à celui-ci comment diviser le mot spécifiquement. Cette situation arrive le plus souvent avec des mots qui comprennent des accents. Par exemple, si le mot "stéréophonique" déborde en fin de ligne, il est possible d'écrire "sté\-réo\-pho\-ni\-que" pour indiquer à LaTeX comment il peut séparer les syllabes.

   When there is an end-of-line overflow caused by a word that LaTeX does not recognize, it is possible to tell LaTeX how to divide the word specifically. This happens most often with words that include accents. For example, if the word "stéréophonique" spills over at the end of the line, you can write "sté\-réo\-pho\-ni\-que" to tell LaTeX how it can separate the syllables.


4.1) Sous Linux / Linux

   Le document LaTeX est compilé à l'aide de la commande "make". Celui-ci est recompilé autant de fois que requis afin d'éliminer les dépendances non résolues. De plus, il prend en charge la compilation de la bibliographie et regénère tous les fichiers dans les répertoires "gnuplot" et "dia".

   The LaTeX document is compiled using the make command and it is recompiled as many times as required to eliminate unresolved dependencies. In addition, it takes care of the compilation of the bibliography and regenerates all files in the "gnuplot" and "dia" directories.

   La commande "make help" affiche toutes les options.
   
   The "make help" command displays all the options.

   N.B. Selon la distribution de Linux utilisée, certains logiciels peuvent devoir être installés manuellement.

   N.B. Depending on the Linux distribution used, some software may have to be installed manually.


4.2) Sous Windows / Windows

   Le gabarit Windows a été développé sous MikTeX 2.8.

   The Windows template has been developed under MikTeX 2.8.

   Il faut compiler seulement le fichier Document.tex car les autres fichier ".tex" sont inclus par ce premier.

   You have to compile only Document.tex, because the other .tex files are included in it.

   ATTENTION: Le format d'encodage de tous les fichier ".tex" est UTF-8.

   WARNING: The encoding format of all .tex files is UTF-8.


5) Visualisation des résultats sous Linux / Visualize results under Linux

   Afin de voir les résultats de la compilation, vous pouvez utiliser la commande XDVI (tapez "make show"). Ceci permet de voir les changements suite à une recompilation.

   To see the results of the compilation, you can use the XDVI command (type "make show"). This allows you to see the changes after a recompilation.
